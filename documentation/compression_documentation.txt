meshTalk — Compression Documentation
====================================

State: current runtime
Date: 2026-04-28

RU
--
1. Назначение
- уменьшить число частей;
- сократить полный wire-size;
- уменьшить число повторов и нагрузку на эфир;
- сохранить точное восстановление исходного текста.

2. Pipeline
1. исходный UTF-8 текст;
2. опциональная нормализация;
3. компрессия;
4. упаковка в relay `DATA` body;
5. фрагментация;
6. шифрование во внешнем `envelope_v3`.

3. Нормализация
Режимы:
- `off`
- `tokens`
- `sp_vocab`
- `auto`

4. Алгоритмы сжатия
- `BYTE_DICT`
- `FIXED_BITS`
- `DEFLATE`
- `ZLIB`
- `BZ2`
- `LZMA`
- `ZSTD`

5. Как выбирается режим
- runtime считает кандидаты по доступным режимам;
- сравнивает не только packed size, но и полный wire-size;
- учитывает overhead relay header и внешнего контейнера;
- сжатие включается только если `wire_comp < wire_plain`.

6. Что хранится в очереди
- `cmp`
- `cmp_norm`
- `cmp_eff_pct`

7. Что видно в логах
- `NORM`
- `COMPRESS`
- `COMPSTAT`

Важно:
- `COMPSTAT top_mode` — агрегированная статистика по сессии, а не режим последнего сообщения.

8. Что видно в UI
- для исходящего сообщения показывается фактический режим этой группы;
- для входящего — конкретный режим из metadata или generic `MC` для сжатого v3 payload.

EN
--
1. Purpose
- reduce fragment count;
- reduce total wire-size;
- reduce retries and RF load;
- preserve exact source-text recovery.

2. Pipeline
1. source UTF-8 text;
2. optional normalization;
3. compression;
4. wrap into relay `DATA` body;
5. fragment;
6. encrypt into outer `envelope_v3`.

3. Normalization
Modes:
- `off`
- `tokens`
- `sp_vocab`
- `auto`

4. Compression codecs
- `BYTE_DICT`
- `FIXED_BITS`
- `DEFLATE`
- `ZLIB`
- `BZ2`
- `LZMA`
- `ZSTD`

5. Selection rule
- runtime evaluates candidates from supported modes;
- compares full wire-size, not just packed payload size;
- includes relay-header and outer-container overhead;
- compression is enabled only when `wire_comp < wire_plain`.

6. Queue metadata
- `cmp`
- `cmp_norm`
- `cmp_eff_pct`

7. Logging
- `NORM`
- `COMPRESS`
- `COMPSTAT`

Important:
- `COMPSTAT top_mode` is a session aggregate, not the codec of the last message.

8. UI presentation
- outgoing messages show the actual codec used for that message group;
- incoming messages show the explicit codec when available, otherwise generic `MC` for compressed v3 payload.
